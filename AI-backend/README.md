# 🧠 TruthLens Backend — Gemini AI with Local Backup System

TruthLens uses a hybrid AI-based detection system to analyze text content and classify it into SAFE, HARMFUL, or FAKE categories. The system is designed to be reliable and always functional by combining two layers of detection: the Gemini AI API as the primary intelligent engine and a locally built keyword-based analyzer as a backup system. This hybrid design ensures that the system continues working even if the external AI service fails due to network errors, API limits, or server downtime.

In the TruthLens workflow, text content is first received from either the Kid Web Extension or the Parent Dashboard web application. This content is sent to the backend API through a POST request to the `/analyze` endpoint. Once the backend receives the text, it attempts to process the input using the Gemini AI model. Gemini acts as the main detection system because it can understand context, analyze natural language, and classify complex sentences with higher accuracy than simple keyword matching. Gemini evaluates the text and returns a structured response containing a classification category such as SAFE, HARMFUL, or FAKE along with a confidence level indicating how certain the model is about the prediction.

However, relying only on an external AI service can introduce risks such as slow response times, internet failures, quota exhaustion, or unexpected API errors. To prevent system failure in such situations, TruthLens includes a built-in local backup analyzer. If Gemini fails to respond or throws an error, the system automatically switches to the local detection logic without interrupting the user experience. This automatic switching is handled using a try-catch mechanism inside the backend code. The try block attempts to process the request using Gemini, and if any failure occurs, the catch block activates the local analyzer to generate a response.

The local analyzer uses predefined datasets containing harmful keywords, harmful intent phrases, and fake content indicators. Harmful keywords include words such as kill, attack, hack, destroy, steal, breach, malware, and phishing. Harmful intent phrases include expressions like "i will attack", "i will hack", "how to hack", "i will steal", and "how to destroy". These phrases indicate stronger intent and are given higher importance during detection. Fake content indicators include phrases commonly found in misleading or scam messages such as "miracle cure", "100% guarantee", "instant results", "secret formula", and "earn money fast". When the local analyzer detects one or more of these patterns in the text, it assigns a score and classifies the message accordingly as HARMFUL or FAKE. If no suspicious patterns are detected, the content is classified as SAFE.

The backend system is built using Node.js and Express.js, which handle incoming requests from frontend components such as the browser extension and dashboard interface. The backend also uses CORS middleware to allow communication between different frontend applications and the server. For secure API key management, the Gemini API key is stored in an environment file (.env) rather than directly inside the source code. This protects sensitive credentials and improves system security.

When a request is made to the `/analyze` endpoint, the client sends a JSON object containing the text to be analyzed. The backend processes the text using Gemini first. If Gemini successfully returns a response, the backend forwards that response back to the frontend along with an indicator showing that the result came from the Gemini AI source. If Gemini fails, the backend returns a result generated by the local backup analyzer and includes a source label indicating that the local system handled the request. This transparency helps developers and users understand which detection system was used.

The Kid Web Extension uses this API to analyze webpage content in real time. For example, it can send visible text from a webpage to the backend, receive the classification result, and display warnings, block unsafe content, or highlight suspicious material. The Parent Dashboard uses the same API to monitor activity, display detection results, and generate safety reports. Because both systems communicate with the same backend endpoint, the architecture remains consistent and scalable.

This hybrid Gemini plus local backup approach offers several advantages. It provides intelligent content analysis using advanced AI reasoning while maintaining system reliability through offline keyword detection. It ensures continuous operation even during service outages and reduces dependency on a single external service. The design also improves performance because the local analyzer can respond quickly when needed. Additionally, the architecture is scalable and can be extended in the future to support more features such as content history logging, user-based analytics, real-time alerts, file scanning, and image content analysis.

Overall, the TruthLens backend is designed as a reliable, production-style AI service that integrates advanced cloud-based intelligence with a strong fallback mechanism. By combining Gemini AI for intelligent detection and a locally built analyzer for backup reliability, the system ensures consistent performance, enhanced safety, and uninterrupted service across all connected frontend components.
